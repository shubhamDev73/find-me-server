Documentation

Root node: /api/

    Authorization end-points. All are POST requests.

  • /register/ : to register. data: username, password. returns "token": auth token
  • /login/ : to login. data: username, password. returns "token": auth token
  • /logout/ : to logout.

    User end-points. Require "Authorization: Bearer <auth-token>" as header. All are GET requests.

  • /me/ : retrieve information about me. "nick", "avatar" (with mood), "avatar-base" (without mood), "personality", "interests", "mood".
  • /interests/ : retrieve all interests. [ {"id", "name"}, {},..].
  • /interests/<interest-id> : retrieve all data for this interest id. {"name", "questions": [ {"id", "text"}, {},..]}.
  • /found/ : retrieve all users connected with me. [ {"nick", "avatar", "retained"}, {},..].

All response are in json format and contain "error" field which contains the error message if an error occured, otherwise empty.

For authorization endpoints, auth token may expire due to security reasons. If this happens, ask user to login again.
